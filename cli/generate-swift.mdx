Generates the type definitions for a given Typesync schema and target platform.

## Usage

```bash
typesync generate --definition <filePathOrPattern> --platform <platform> --outFile <filePath> --indentation <indentation> --customPydanticBase <classImportPath> --debug <debug>
```

<ParamField type="string" path="definition" required>
The exact path or a Glob pattern to the definition file or files. Each definition file must be a YAML or JSON file containing model definitions.

- Example single file path: `definition/models.yml`
- Example Glob pattern: `'definition/**/*.yml'`

</ParamField>

<ParamField type="string" path="platform" required>
  Target platform and version. The available platforms are:

- `ts:firebase-admin:12`
- `ts:firebase-admin:11`
- `ts:firebase:10`
- `ts:firebase:9`
- `swift:firebase:10`
- `py:firebase-admin:6`

</ParamField>

<ParamField type="string" path="outFile" required>
  The path to the output file.
</ParamField>

<ParamField type="int" path="indentation" default={2}>
  Indentation or tab width for the generated code.
</ParamField>

<ParamField type="string" path="customPydanticBase">
  The base Pydantic class from which all the generated Python models will extend. Must be in the format `x.y.ModelName`. If this is not provided, the generated models will extend from `pydantic.BaseModel`.

Example values:

- `custom.MyModel`
- `a.b.c.MyCustomModel`

</ParamField>

<ParamField type="boolean" path="debug" default={false}>
  Whether to enable debug logs.
</ParamField>
